Hosted FAQ

14-day free trial

You can use the service for free during the 14 day trial. No credit card required to sign up. Cancel at any time.

What is "concurrent job"?

Every push to the repository starts a new build with a single or multiple jobs. If your current plan includes only 1 concurrent job and there is another build job currently running all consequent jobs will be queued until the first job finishes. With multiple concurrent jobs your team will be getting results faster or you can use build matrix and parallel testing to build and test, for instance, Debug/Release or x64/x86 configurations in parallel.

What is build cache?

AppVeyor runs every build on a clean virtual machine. Virtual machine state is not preserved between builds which means every build downloads sources, installs NuGet packages, Node.js modules, Ruby gems or pulls dependencies. Build cache allows you to preserve the contents of selected directories and files between project builds.

Are there any build time restrictions?

All plans have maximum build job execution time of 60 minutes.

Enterprise FAQ

30-day free trial

During the 30-day trial period we will help you with the installation and setting up AppVeyor on your premises to make sure it works great for your team.

What are "unlimited concurrent jobs and build agents"?

Unlike hosted AppVeyor, Enterprise edition is priced for the number of users working with it while the number of concurrent jobs and build agents (servers or clouds) is not limited.

Having unlimited concurrent jobs and build agents allows cost-effective testing of complex software in a cloud. Check how one of our customers uses AppVeyor to test their software on Azure.

These jobs run on your own cloud such as Azure, AWS EC2, GCE, Hyper-V or even a server under your desk and you pay those providers separately. AppVeyor makes sure cloud VM is terminated once build job is finished.

What do I need to give AppVeyor Enterprise a try?

You need at least 2 CPU/7 GB RAM VM with Windows Server 2012 R2 or above to install AppVeyor Enterprise and its dependencies and an active Azure/AWS/Google cloud subscription or Hyper-V server to provision build job VMs. For more details please see AppVeyor Enterprise planning guide.
